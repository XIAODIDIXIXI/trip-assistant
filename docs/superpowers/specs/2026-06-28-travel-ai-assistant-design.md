# 旅游 AI 助手 — 产品设计文档

> 日期：2026-06-28 | 状态：已确认

## 1. 项目概述

**产品定位**：旅游 AI 助手 Web 应用，帮助用户智能规划旅行行程、探索目的地、记录旅行足迹。

**目标用户**：想轻松规划旅行的中国用户。

**技术栈**：
- 前端：Vue3 + Vite + Element Plus + Vue Router 4
- 后端：Spring Boot + MySQL
- AI：DeepSeek API
- 地图：高德地图 JSAPI
- 3D 地球：Three.js + Globe.GL
- 设计：Impeccable + Taste Skill + frontend-design

## 3. 页面详细设计

### 3.1 首页 — 旅行指挥中心

**定位**：行程规划的起点，输入表单联动全页。

**结构**（从上到下）：
1. **规划面板** — 目的地搜索选择器 + 天数 + 预算 + 出行类型（家庭/情侣/独自/朋友）+ 兴趣偏好（美食/自然/人文/购物/户外）+ 生成行程按钮
2. **数据卡片行** — 天气卡片、预算预估卡片、已点亮城市数、最近行程摘要
3. **热门目的地** — 城市标签云，点击可快速填入目的地
4. **生成结果区**（表单提交后展开）— 地图区域（高德地图标注景点+路线）+ AI行程卡片（Day1/Day2/Day3...）

**交互逻辑**：
```
表单填写 → 点击"生成行程"
  → 天气卡片自动刷新目的地天气
  → 地图区域标注景点坐标 + 画出行路线
  → AI流式生成每日行程
  → 预算卡片更新预估
```

**数据流**：`POST /api/trips/generate` → 后端调 DeepSeek + 高德 → 返回 { 天气, 景点[], 每日行程[], 预算, 地图坐标 }

### 3.2 对话页 — AI 旅游助手

**定位**：旅游专属 ChatGPT，聊天中枢，所有能力都在对话里可达。

**结构**：
- **左侧（80%）**：聊天区，消息列表 + 底部输入框
- **右侧（20%）**：历史会话列表 + 新建对话按钮
- **顶部**：当前行程上下文横幅（从首页联动过来，无行程时显示快捷提问）

**核心能力**：
1. 流式输出 — AI 回复逐字渲染
2. 上下文记忆 — 持久化会话历史
3. 当前行程联动 — 自动感知首页生成的行程
4. 快捷提问 — 无行程时引导用户
5. 嵌入卡片 — 行程卡片、地图卡片直接嵌在消息里
6. 反馈 — 每条 AI 回复 👍👎
7. 收藏 — 点消息旁的收藏存入"我的"

**数据流**：`POST /api/chat/send`（SSE流式）→ 后端调 DeepSeek（带function calling）→ 流式返回

### 3.3 足迹页 — 3D 地球足迹

**定位**：个人旅行足迹可视化，去过哪里点亮哪里。

**结构**：
- **全屏 3D 地球**（上半部分）— Three.js 渲染，拖拽旋转，滚轮缩放
- **足迹列表**（下半部分）— 时间线排列，城市/日期/行程链接

**交互**：
- 鼠标拖拽 = 旋转地球
- 滚轮 = 缩放
- 点击光点 = 弹出城市卡片（城市名/日期/当时行程）
- 点列表项 = 地球自动旋转到对应位置

**数据流**：`GET /api/footprints` → 返回用户去过的城市列表 { city, lat, lng, date, tripId }

### 3.4 我的页 — 个人中心

**结构**（从上到下）：
1. **个人信息区** — 头像、昵称、手机号、编辑资料按钮
2. **数据概览** — 行程数/点亮城市数/会话数/收藏数
3. **迷你足迹地图** — 静态缩略图，点击跳转到足迹页
4. **成就徽章** — 打卡5城/预算达人/说走就走等勋章
5. **我的行程列表** — 历史生成的行程，可查看详情
6. **历史会话** — 对话页的会话记录，点击继续聊
7. **浏览过的城市** — 城市标签云

## 5. 后端模块

| 模块 | 职责 |
|------|------|

## 6. 数据模型概要

```
User        用户      id, phone, nickname, avatar
Trip        行程      id, userId, city, days, budget, type, preferences, content(JSON)
ChatSession 会话      id, userId, title, createTime
ChatMessage 消息      id, sessionId, role, content, type
Footprint   足迹      id, userId, city, lat, lng, date, tripId
Collection  收藏      id, userId, type, targetId
BrowsedCity 浏览城市   id, userId, city, time
```

## 7. 非功能需求

- 流式输出：使用 SSE，延迟 < 500ms 首字
- 移动端：响应式，侧边栏在窄屏收起
- 地图：高德 JSAPI 2.0，安全密钥配置
- 3D 地球：低端设备降级为静态图

## 9. 设计决策记录

- **导航布局选集中侧边栏** — 4个入口，清晰不臃肿
- **首页用联动模式** — 表单是发动机，其他模块是仪表盘
- **对话有行程上下文横幅** — 首页规划后跳对话，AI 自动感知
- **足迹放独立页** — 全屏 3D 沉浸体验，侧边栏独立入口
- **登录先极简** — 手机号验证码，后续补其他方式
